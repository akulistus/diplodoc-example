# Модуль генератора сигналов

Данный модуль используется в примерах для моделирования работы графика с сигналами,
изменяющимися в реальном времени.

Исходный код модуля располагается по [ссылке](https://github.com/IncartDev/signal-generator-js).

## Установка

Модуль устанавливается как git submodule. Рекомендуется производить установку в директорию `src/submodules`.

```sh
  $ git submodule add git@github.com:IncartDev/signal-generator-js.git
```

## Файл конфигурации
Файл конфигурации требуется для установки режимов работы генератора сигналов. Конфигурация описывается в файле формата json
и состоит из трех основных свойств.
* `pointBlockSize` - \{**int**\} количество точек в блоке, передающихся графическому полотну для отрисовки. Обычно указывается равным 40.
* `freq` - \{**int**\} частота генерации точек в единицу времени. Обычно указывается равным 1000.
* `frames` - \{**array**\} содержит описания различных каналов. Обычно используют один кадр с различными каналами.

```json
{
  "pointBlockSize": 40,
  "freq": 1000,
  "frames":[
    {
      "channels":[]
    }
  ]
}
```
### Свойство channel

```json
{
  "channels": [
    {
      "id": 0,
      "name": "ecg",
      "state": "on",
      "maxSignalCount": 8,
      "totalSignals": 8,
      "freq": 1000,
      "divFreq": 1,
      "unit": "mkV",
      "normalValue": 1000,
      "maxValue": 300000,
      "minValue": -300000,
      "allowDivsFreq": [1, 4],
      "allowFreqs": [1000, 250],
      "allowCountsSignal": [0, 1, 3, 8],
      "currChannelConfiguration": "3x5",

      "clbK": [800, 800, 800, 800, 800, 800, 800, 800],
      "clbB": [0, 0, 0, 0, 0, 0, 0, 0],
      "clbPtnADC_A": [0, 0, 0, 0, 0, 0, 0, 0],
      "clbPtnADC_B": [12, 12, 12, 12, 12, 12, 12, 12],

      "channelConfigurations":[],
    }
  ]
} 
```
Каждый канал описывается рядом следующих свойств:
* `id` - \{**int**\} уникальный идентификатор канала.
* `name` - \{**string**\} уникальное? имя канала. Используется для привязки канала генератора сигнала к каналу графического полотна.
* `state` - \{**"on"** или **"off"**\} состояние канала, включен\(производится генерация\) или выключен\(генерация не производится\), соответственно.
* `maxSignalCount` - \{**int**\} максимально количество сигналов в одной конфигурации.
* `totalSignals` - \{**int**\} количество различных сигналов, присутствующих в данном канале.
* `freq` - \{**int**\} частота сигнала \(главная частота\). Обычно указывается равным 1000.
* `divFreq` - \{**int > 0**\} делитель главной частоты. Обычно указывается равным 1.
* `unit` - \{**string**\} единицы измерения. Например, `mkV`\(микровольты\) для ЭКГ. 
* `normalValue` - \{**int > 0**\} - среднее значение чего?
* `maxValue` - \{**int > 0**\} - максимальное значение чего?
* `minValue` - \{**int > 0**\} - минимальное значение чего?
* `allowDivsFreq` - \{**array**\} разрешенные значения делителя главной частоты. Например, \[1, 8\].
* `allowFreqs` - \{**array**\} разрешенные частоты, получающиеся после применения делителя частоты. Например, \[1000, 125\].
* `allowCountsSignal` - 
* `currChannelConfiguration` - \{**string**\} конфигурация канала, которая используется для генерации в данный момент. Выбирается из списка конфигураций,
указанных в `channelConfigurations`.
* `channelConfigurations` - \{**array**\} содержит описания возможных конфигураций сигналов для данного канала. \(Далее подробнее\)

Существует ряд сервисных свойств, значения которых, в общем случае \(для тестовых сигналов\), не будут меняться. В следующих свойствах:
clb - калибровка.

* `clbB` - калибровочный коэффициент b. Из линейной функции y = kx + **b**.

### Свойство channelConfigurations
`channelConfigurations` содержит описания возможных конфигураций сигналов для данного канала.
```json
  {
    "channelConfigurations": [
      { "name": "3x5", "signals": [] }
    ]
  }
```
* `name` - \{**string**\} название конфигурации.
* `signals` - \{**array**\} содержит описания сигналов, которые генерирует данная конфигурация.
### Свойство signals
```json
{
  "signals": [
    { "id": 0, "name": "ECG0", "generator": {} },
    { "id": 1, "name": "ECG1", "generator": {} }
  ]
}
```
* `id` - \{**int**\} уникальный идентификатор сигнала.
* `name` - \{**string**\} уникальное? имя сигнала.
* `generator` - \{**object**\} задает параметры генерируемого сигнала.
### Свойство generator
```json
{
  "generator": {
    "type": "sin",
    "ampl": 0.5,
    "perion": 200,
    "shift": 5
  }
}
```
* `type` - \{**"sin"** или **"rect"**\} тип генерируемого сигнала.
* `ampl` - \{**int** или **float**\} амплитуда сигнала в единицах отсчета АЦП.
* `period` - \{**int**\} период сигнала в миллисекундах.
* `shift` - \{**int**\} смещение по оси y.

## Подключение генератора сигналов
Импорт и инициализацию генератора сигнала рекомендуется производить в файл инициализации графического полотна.
```js
// Импорт объекта генератора сигнала.
import { SignalGenerator } from "<путь до подмодуля>/signal-generator-js/signal-generator";

// Внутри функции инициализации графического полотна
{
  // Создание экземпляра класса генератора сигналов
  const generator = new SignalGenerator();

  // Загрузка конфигурации генератора сигналов
  generator.setSettings(SignalGenerator);
  generator.onFrameIsReady = message => {
    visualizatorProvider.pushAdcMessage(message, 1372205);
    visualizatorProvider.pushMessage(message);
  }

  // Запуск генератора сигнала
  generator.start();
}
```

  [Пример](getting-started) использования генератора сигнала.

{% endnote %}

## Методы
* `setSettings(settings: string)` - метод, загружающий конфигурацию в созданный ранее класс.
* `onFrameIsReady(handler: func)` - эвент, срабатывающий в момент готовности блока данных. Эвент передаёт подготовленный блок данных в хэндлер.
* `getChannelPointCounter(frameIndex: int, channelIndex: int)` - метод, позволяющий получить количесто сгенерировнных точек опр. канала, где 
`frameIndex` - индекс кадра, а `channelIndex` - индекс канала в кадре.
* `start()` - запуск генерации сигналов.
* `stop()` - завершение генерации сигналов.
