# Свойства объекта view

Простейшая конфигурация выглядит следующим образом:

```json
{
  "view": {
    "canvas": {
      "items": [
        {
          "type": ...,
          "channels": [ ... ]
        }
      ]
    }
  }
}
```

Описание поля `view` может быть расширено:

```json
{
  "view": {
    "canvas": {
      "items": [
        {
          "type": "plot:online:base",
          "sectors": [
            { "name": "right", "width": "10 px" },
            { "name": "top", "height": "30 px" },
            { "name": "bottom", "height": "10 px" }
          ],
          "content": [{ "name": "grid" }, { "name": "border" }],
          "channels": [
            {
              "id_channel": 0,
              "showIsolines": true,
              "y": "46 %",
              "height": "54 %",
              "scaleX": { ... },
              "scaleY": { ... },
              "variants": [
                {
                  "name": "1x3",
                  "isolines": [{ ... }],
                  "signals": [{ ... }],
                  "content": [{ ... }]
                }
              ]
            },
          ]
        }
      ]
    }
  }
}
```

| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `type` | **string** | *ОБЯЗАТЕЛЬНОЕ* | Тип отображаемого графика. Возможные [виды графиков](graphs). |
| `sectors` | **Array\<object\>** | Только центральная область | Содержит объекты, определяющие дополнительные области графического полотна. Области могут быть использованы для отображения различных элементов интерфейса. |
| `content` | **Array\<object\>** | &mdash; | Содержит дополнительные графические элементы, отображаемые на графическом полотне. |
| `channels` | **Array\<object\>** | &mdash; | Содержит структуру, связывающую сигналы из `data` с графическии полотном. |

## Поле sectors
`sectors` определяет дополнительные области графического полотна. Области могут быть использованы для отображения различных элементов интерфейса.
Существуют следующие области:
* top
* topCentral
* bottom
* bottomCentral
* left
* leftCentral
* central
* right
* rightCentral

Пример расположения областей на полотне представлена далее.
<img src="/_images/sectors.png" width="100%"/>

```json 
{
  "sectors": [
    { "name": "right", "width": "10 px" },
    { "name": "top", "height": "30 px" },
    { "name": "bottom", "height": "10 px" }
  ]
}
```
| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `name` | **string** | *ОБЯЗАТЕЛЬНОЕ* | Тип определяемой области. |
| `width`\\`height` | **string** | 0 px | Размер соответствующей области. Если область располагается вертикально &mdash; `width`. Горизонтально &mdash; `height`. |

Область может быть заполнена различнми графическими элементами.

## Поле content

`content` содержит дополнительные графические элементы, отображаемые на графическом полотне.

```json
{
  "content": [{ "name": "grid" }, { "name": "border" }]
}
```
{% note info %}

  Подробнее о графических элементах можно прочитаь в статье [Графические элементы](graph-elements).

{% endnote %}


## Поле channels

`channels` содержит структуру, связывающую сигналы из `data` с графическии полотном.

```json
{
  "channels": [
    {
      "id_channel": 0,
      "showIsolines": true,
      "isSynchronizedScaleX": true,
      "y": "46 %",
      "height": "54 %",
      "scaleX": { ... },
      "scaleY": { ... },
      "variants": [
        {
          "name": "1x3",
          "isolines": [{ ... }],
          "signals": [{ ... }],
          "content": [{ ... }]
        }
      ]
    },
  ]
}
```

| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `id_channel` | **number** | Задаётся неявно и равно индексу в данном массиве `channels` | Идентификатор канала, определённого в `data:{  channels: [ ... ] }`. |
| `showIsolines` | **boolean** | false | Отображает изолиннии на графическом полотне. |
| `isSynchronizedScaleX` | **boolean** | true | Синхронизирует отрисовку сигналов конфигурируемого канала. |
| `y` | **string** | 0 % | Положение по оси Y левой верхней точки объекта. Направление по оси Y (для каналов) - сверху вниз. Y рассчитывается относительно области отрисовки сигналов. |
| `height` | **string** | 100 % | Устанавливает пространство по оси `y`, занимаемое одним каналом. |
| `scaleX` | **object** | *КАКОЕ?* | Задаёт масшаб сигнала по оси X. |
| `scaleY` | **object** | *КАКОЕ?* | Задаёт масшаб сигнала по оси Y. |
| `variants` | **Array\<object\>** | *ОБЯЗАТЕЛЬНОЕ* | Содержит описания вариантов групп сигналов. Для сигнала, имеющего одну конфигурцию, определение вариантов допускается опустить. См. [Файл конфигурации](config-file) |

### scaleX
`scaleX` задаёт масшаб сигнала по оси X.
```json
{
  "scaleX": {
    "defaultValues": [2.5, 5, 10, 20, 40],
    "defaultIndex": 2,
    "distanceUnit": 0,
    "timeUnit": 2
  }
}
```

| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `defaultValues` | **Array\<number\>** | [10] | Задаёт массив масштабирующих значений для соответствующей оси. |
| `defaultIndex` | **number** | 0 | Индекс выбранного масштабирующего значения. |
| `distanceUnit` | **number** | &mdash; | Задаёт единицу измерения длины для подписи оси. Значение должно равняться индексу одной из определенных масштабирующих единиц измерения. См. [Поля объекта data](data-propertis). |
| `timeUnit` | **number** | &mdash; | Задаёт единицу измерения времени для подписи оси. Значение должно равняться индексу одной из определенных масштабирующих единиц измерения. См. [Поля объекта data](data-propertis). |

### scaleY
Данные поля определяют масшатб сигнала по осям Y и X соответственно. Возможные параметры конфигурации рассмотрим на примере поля `scaleY`.
```json
{
  "scaleY": {
    "defaultValues": [2.5, 5, 10, 20, 40],
    "defaultIndex": 2,
    "distanceUnit": 0,
    "valueUnit": 2
  }
}
```

| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `defaultValues` | **Array\<number\>** | [5] | Задаёт массив масштабирующих значений для соответствующей оси. |
| `defaultIndex` | **number** | 0 | Индекс выбранного масштабирующего значения. |
| `distanceUnit` | **number** | &mdash; | Задаёт единицу измерения длины для подписи оси. Значение должно равняться индексу одной из определенных масштабирующих единиц измерения. См. [Поля объекта data](data-propertis). |
| `valueUnit` | **number** | &mdash; | Задаёт единицу измерения величины для подписи оси. Значение должно равняться индексу одной из определенных масштабирующих единиц измерения. См. [Поля объекта data](data-propertis). |

### variants

`variants` содержит описания вариантов групп сигналов.

```json
{
  "variants": [
    {
      "name": "1x3",
      "isolines": [{ ... }],
      "signals": [{ ... }],
      "content": [{ ... }]
    }
  ]
}
```

| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `name` | **string** | *ОБЯЗАТЕЛЬНОЕ* | Название варианта. Должно совпадать с одним из названий варианта выбранного канала. |
| `isolines` | **Array\<object\>** |  | Содержит конфигурации вариантов изолиний конфигурируемого варианта канала. Количество изолиний должно равняться количеству сигналов конфигуруемого варианта. |
| `signals` | **Array\<object\>** |  | Содержит конфигурации отображения сигналов конфигурируемого варианта. |
| `content` | **Array\<object\>** |  | Содержит дополнительные графические элементы сигнала, отображаемые на графическом полотне. Подробнее о [графических элементах сигнала](graph-elemetns). |

#### isolines

`isolines` содержит конфигурации вариантов изолиний конфигурируемого варианта канала. Количество изолиний должно равняться количеству сигналов конфигуруемого варианта.

```json
{
  "isolines": [
    {
      "id": 0,
      "y": "50 %",
      "color": "#3f48cc"
    }
  ]
}
```

| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `id` | **number** | Поле задаётся неявно и равно индексу в массиве `isolines` | Уникальный идентификатор созданной изолиннии. |
| `y` | **string** | Рассчитвается и устанавливается на уровне среденго значения отображаемого сигнала | Положение по оси Y. |
| `color` | **string** | #000000 | Цвет изолинии на графике. |

#### signals

`signal` содержит конфигурации отображения сигналов конфигурируемого варианта.

```json
{
  "signals": [
    {
      "id": 0,
      "type": "line",
      "fillType": "period",
      "color": [179, 128, 0, 255] ,
      "id_isoline": 0,
      "pointCount": 1,
      "title": "signal",
    }
  ]
}
```

| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `id` | **number** | Поле задаётся неявно и равно индексу в массиве `signals` | Уникальный идентификатор созданной сигнала. |
| `type` | **string** | line | Вид отображаемого сигнала, например, `line` или `histogram`. |
| `fillType` | **string** | period | Тип заполнения. |
| `color` | **string** | #000000 | Цвет изолинии на графике. |
| `id_isoline` | **number** | Поле задаётся неявно и равно индексу в массиве `signals`. | Идентификатор изолинии конфигурируемого сигнала.  Связывает сигнал с изолинией, определенной в масстве `isolines`. |
| `pointCount` | **number** | ? | ? |
| `title` | **string** | &mdash; | Название сигнала. |
