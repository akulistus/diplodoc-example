# Файл конфигурации

График на графическом полотне строится на основе файла конфигурации. 
Он позволяет устанавливать отслеживаемые сигналы и гибко управлять их отображением.

Файл конфигурации имеет формат .json и состоит из двух основных частей: 
* data &mdash; описание входных сигналов.
* view &mdash; описание отображаемых графиков.

В упрощённом виде выглядит следующим образом:
```json
{
  "data": {...},
  "view": {...}
}
```
## Объект data

```json {2}
{
  "data": {...},
  "view": {...}
}
```
`data` описывает данные и источники данных\(сигналов\), которые используются для построения графиков и отображения их на графическом полотне.
`data` в простейшем случае состоит из двух элементов: `units` и `source`.
```json
{
  "data": {
    "units": {...},
    "channels": [...],
  }
}
```

### Units
В `units` задаются масштабирующие единицы измерения, которые могут использоваться для пересчета данных сигнала в соответствующий масштаб 
и подписи осей. Ниже приведен пример задания милливольта в качестве единицы измерения:
```json
{
  "units": [
    {
      "text": "mV",
      "scale": 1
    }
  ]
}
```
| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `id` | **number** | Задаётся неявно и равно индексу в массиве `units` | Уникальный идентификатор созданной единицы измерения. Требуется для связки сигнала единицы измерения (разбирается далее). |
| `text` | **string** | "" | Название единицы измерения. Нужно для корректной подписи осей графика. |
| `scale` | **number** | 1 | Коэффициент позволяет выразить отсчёты сигнала в соответствующих единицах. Влияет масштаб сигнала на графике. |

### Channels
`channels` является массивом каналов, содержащих отсчеты различных сигналов.

`channels` имеет следующий структурный вид:
```json
{
  "channels": [
    {
      "name": "ecg",
      "signals": 5
    }
  ]
}
```

| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `name` | **string** | *ОБЯЗАТЕЛЬНОЕ* | Уникальное имя канала данных. **Должно совпадать** с именем передающегося канала данных \(подробнее, смотри [Общие сведения](общие-сведения)\). |
| `signals` | **number** | &mdash; | Задаёт количество сигналов в канале. |
| `frequency` | **number** | 1000 | Частота дискретизации сигнала. |

{% note info %}

  Подробнее о [Каналах](general-info).

{% endnote %}

## Объект view

```json {3}
{
  "data": {...},
  "view": {...}
}
```

`view` описывает конфигурацию графического полотна: тип графика, их количество, дополнительные элементы.
```json
{
  "view": {
    "canvas": {
      "items": [
        {
          "type": "plot:online:base",
          "channels": [
            {
              "signals": [{ "color": "#026ce5" }]
            }
          ]
        }
      ]
    }
  }
}
```
Существует несколько объектов типа `plot...`, но в данной статье будет рассмотрен `plot:online:base`.

{% note info %}

  Подробнее о графика можно прочитать в статье [Графики](graphs).

{% endnote %}

| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `type` | **string** | *ОБЯЗАТЕЛЬНОЕ* | Тип отображаемого графика. |
| `channels` | **Array\<object\>** | &mdash; | Содержит структуру, связывающую сигналы из `data` с графическии полотном. |

Для связывания сигналов с графическим полотном используются `id` сигналов, укаызываемые в `data: \{ channels: [ ... ] \}`. Если `id` не было присвоено явно, то связка сигнала с графическим полтоном происходит по индексу сигнала в массиве `data: \{ channels: [ ... ] \}`.

{% note info %}

  Подробнее о возможных конфигурациях этих свойств можно узнать в статьях [Графические элементы](graph-elements) и [Графики](graphs).

{% endnote %}
