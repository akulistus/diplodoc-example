# Свойства объекта data

Простейшая конфигурация выглядит следующим образом:

```json
{
  "data": {
    "units": {...},
    "channels": [...],
  }
}
```
Описание поля `data` может быть расширено:

```json
{
  "data": {
    "units": [
      { "id": 0, "text": "мм", "scale": 1 },
      { "id": 1, "text": "с", "scale": 1 },
    ],
    "marks": [{ "id": 0, "name": "RR" }],
    "channels": [
      {
        "id": 0,
        "name": "ecg",
        "unit": { "id_unit": 0 },
        "maxSignalCount": 8,
        "frequency": 1000,
        "currVariant": "12x10",
        "variants": [
          { "name": "1x3", "signals": 1 },
          { "name": "3x5", "signals": 3 },
          { "name": "3x7", "signals": 3 },
          { "name": "12x10", "signals": 8 }
        ]
      }
    ]
  },
}
```

## Поле units
В `units` задаются масштабирующие единицы измерения, которые могут использоваться для пересчета данных сигнала в соответствующий масштаб 
и подписи осей. Ниже приведен пример задания милливольта в качестве единицы измерения:
```json
{
  "units": [
    {
      "text": "mV",
      "scale": 1
    }
  ]
}
```
| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `id` | **number** | Поле `id` задаётся неявно и равно индексу в массиве `units` | Уникальный идентификатор созданной единицы измерения. Требуется для связки сигнала единицы измерения (разбирается далее). |
| `text` | **string** | "" | Название единицы измерения. Нужно для корректной подписи осей графика. |
| `scale` | **number** | 1 | Коэффициент позволяет выразить отсчёты сигнала в соответствующих единицах. Влияет масштаб сигнала на графике. |


## Поле marks
`marks` является массивом источников, содержащих данные меток сигналов. 
```json
{
  "marks": [
    { "id": 0, "name": "RR" }
  ],
}
```
| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :---: | --- |
| `id` | **number** | Поле `id` задаётся неявно и равно индексу в массиве `marks` | Уникальный идентификатор созданной единицы измерения. Требуется для связки сигнала c метками. |
| `name` | **string** | *ОБЯЗАТЕЛЬНОЕ* | Уникальное имя источника данных меток. **Должно совпадать** с именем передающегося источника данных меток \(подробнее, смотри [Общие сведения](общие-сведения)\). |


## Поле channels
`channels` является массивом каналов, содержащих отсчеты различных сигналов.

```json
{
  "channels": [
    {
      "id": 0,
      "name": "ecg",
      "unit": { "id_unit": 0, "scale": 0.001 },
      "maxSignalCount": 8,
      "frequency": 1000,
      "currVariant": "12x10",
      "variants": [
        { "name": "1x3", "signals": 1 },
        { "name": "3x5", "signals": 3 },
        { "name": "3x7", "signals": 3 },
        { "name": "12x10", "signals": 8 }
      ]
    },
  ]
}
```
| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :--- | :--- |
| `id` | **number** | Поле задаётся неявно и равно индексу в массиве `marks` | Уникальный идентификатор созданной единицы измерения. Требуется для связки сигнала c метками. |
| `name` | **string** | *ОБЯЗАТЕЛЬНОЕ* | Уникальное имя источника данных меток. **Должно совпадать** с именем передающегося источника данных меток \(подробнее, смотри [Общие сведения](общие-сведения)\). |
| `unit` | **object** | Значение первой масштабирующие единицы, определенной в поле `units`. | Устанавливает масштабирующую единицу для сигналов канала. |
| `maxSignalCount` | **number** | Равен занчению поля `signals`, если оно определено. | Максимально возможное кол-во сигналов для данного канала. **Пока в расчетах не используется**. |
| `frequency` | **number** | 1000 | Частота дискретизации сигнала. |
| `currVariant` | **string** | Поле задаётся неявно. Если в конфигурации присутствует поле `variants`, тогда выбирается первый из определенных вариантов. Если вместо `variants` определено поле `signals`, то выбирается оно.| Имя выбранного варианта отображения графиков. Название **должно совпадать** с названием одного из вариантов, указанных в `variants`. |
| `variants` | **Array\<object\>** | &mdash; | Содержит описания вариантов групп сигналов. Поле с одним вариантом может быть заменено полем `signals`. См. [Файл конфигурации](config-file). |

### Поле unit
`unit` устанавливает масштабирующую единицу для сигналов канала.

```json
{
  "unit": {
    "id_unit": 0,
    "scale": 0.001
  }
}
```

| Название | Тип | Значение поумолчанию | Описание |
| :---: | :---: | :--- | :--- |
| `unit_id` | **number** | *ОБЯЗАТЕЛЬНОЕ* | Уникальный идентификатор созданной масштабирующей единицы. |
| `scale` | **number** | Занчение выбранной масштабирующей единицы | Позволяет переопределить коэффициент масштабирующей единицы. |
